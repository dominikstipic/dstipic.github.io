Linux Scheduler - 13/04/2024

During work there is often a need to create scheduled jobs which are started periodically. Linux provides us with the crontab utility whose purpose is to schedule jobs in the Linux. If we type the crontab -e command in our command line we will get the configuration file where we can define our schedule. The schedule is defined by 5 parameters and they are:
  • minute
  • hour
  • day of month
  • month
  • day in week
So for example, if we want to define a python job job.py to run every day at 12:00 during workdays we will put the following line in the crontab schedule configuration file:

0 12 * 1-5 job.py

The crontab.guru website could be used for help when defining our schedules. The cron configuration file can be inspected with the crontab -l command. Executions of our cron tasks can be found on Linux syslog which is located in the /var/log/syslog file. If we type the command cat /var/log/syslog | grep CRON we can locate the following line in our syslog file:

Apr 13 09:32:01 doms-Aspire-A515-45 CRON[206541]: (doms) CMD (job.py)
